How It Works

Three steps. Zero guesswork.

Select a frame, describe user intent, and let AI generate WCAG-aligned accessibility annotations directly inside Figma.

Figma Plugin

Designer selects a UI frame and describes what the user is trying to accomplish

MCP Connector

Request routed through an authenticated, JWT-secured proxy server

Circuit AI

LLM analyzes the frame and generates WCAG 2.1 AA accessibility annotations

Zero API keys on the client. Enterprise-grade JWT/JWKS authentication throughout.

For Designers

Your workflow, supercharged

No context switching. No separate audit tools. Everything happens inside Figma, right where you design.

1

Select any frame

Pick a component, screen, or flow in your Figma file. Works with auto-layout, variants, and nested instances.

2

Describe the user intent

Tell the tool what the user is trying to do — “submit a form,” “navigate settings,” “review a dashboard.” Context makes AI smarter.

3

Flag known risks

Check off concerns you already suspect — color contrast, keyboard navigation, screen reader flow — so the analysis is focused, not generic.

4

Get actionable specs

AI returns WCAG-aligned annotations: accessible names, roles, states, contrast ratios, focus order — ready to paste into your design or hand off to eng.

Under 30 seconds per frame
Zero terminal or code needed
WCAG 2.1 AA compliant output
